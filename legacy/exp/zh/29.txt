There is a multiple-choice question: Which of the following is NOT a reasonable justification for choosing to busy-wait on an asynchronous event?. The options are: A. The wait is expected to be short., B. A busy-wait loop is easier to code than an interrupt handler., C. There is no other work for the processor to do., D. The program executes on a time-sharing system.. Why the correct answer is: B. A busy-wait loop is easier to code than an interrupt handler.?
